Curation Lab — UX 개선 리포트

각 페이지의 폰트 가독성, 하부 페이지 상단 버튼·메뉴, 분석 완료 페이지의 이미지 상태(깨짐) 세 축을 UX 관점에서 점검하고 개선했습니다. 실행 중인 앱을 실제 데이터(완료된 newface_v1 작업, 197장 입력 / 30장 유지, 라이브 A6000 GPU 2장)로 렌더해 캡처했습니다.

날짜 · 2026-07-21 브랜치 · ux/legibility-images-topbar 스택 · Next.js 15 · App Router · Tailwind 뷰포트 · 1440px (+ 560px 좁은 화면)

캡처 방식: 원본/개선 두 빌드를 각각 구동 후 headless Chromium(2x DPR, 다크)으로 동일 스크립트 캡처. 스크린샷의 데이터셋 이미지는 사용자 소유 자료이므로 외부에 게시하지 않고 리포지토리 내 로컬 문서로만 보관합니다.

🔴 이미지 깨짐의 근본 원인 (환경)
현재 실환경에서 모든 이미지가 깨져 있었습니다. DB에 저장된 절대경로가 리팩터 이전 위치(curation/runs/…)를 가리키는데, 소스가 app/ 아래로 이동(curation/app/curation/runs/…)하면서 경로가 어긋나 API가 404를 반환합니다. 파일 자체는 존재합니다.
이번 브랜치의 코드 개선(플레이스홀더 폴백)은 파일이 없을 때 UX가 무너지지 않도록 하는 방어입니다. 실제 이미지를 다시 보이게 하려면 저장 경로 정합성 수정이 필요합니다 — 운영 임시조치로 호환 심링크(curation/runs → curation/app/curation/runs)를 적용해 아래 개선 스크린샷의 썸네일을 복원했습니다. 근본 해결(잡 생성 시 상대경로 저장 또는 DB 마이그레이션)은 별도 권장 항목입니다.

1. 분석 완료 페이지 — 이미지 상태 / 깨짐

리뷰 갤러리 before → after

BEFORE — 전 썸네일 깨진 아이콘 + 파일명만
AFTER — 썸네일 복원 · 로고 아이콘 · TopBar 정돈 · 60장 페이징

※ AFTER 갤러리의 썸네일 사진은 개인정보 보호를 위해 블러 처리했습니다(데이터셋 이미지 비공개). UI·배지·점수·레이아웃은 실제 그대로입니다.

  • 깨진 파일은 이제 깨진 아이콘 대신 플레이스홀더로 처리(아래 모달 참조), 로딩 중에는 스피너.
  • 초기 렌더를 60장으로 제한하고 "더 보기"로 확장 — 문서 높이 10,652px → 3,576px.
  • keep/reject 토글 버튼 24px → 32px, aria-label 및 포커스 링 추가.

이미지 상세 모달 — 깨짐 방어의 핵심 before → after

BEFORE — 깨진 이미지 아이콘 + alt 텍스트
AFTER — "이미지 없음 / 이동됨" 플레이스홀더 + 파일명

이 모달은 원본 전체 해상도를 로드합니다. 해당 이미지는 리젝트되어 newface_v1_rejected/로 이동된 상태라 원본이 실제로 없습니다. BEFORE는 깨진 글리프를 그대로 노출하지만, AFTER는 파일이 없어도 우아하게 폴백합니다 — 즉 개선 효과가 데이터 존재 여부와 무관하게 입증됩니다.

2. 하부 페이지 상단 버튼 · 메뉴

좁은 폭에서의 TopBar 붕괴 560px · before → after

BEFORE — 제목 2줄, "갤러리 리뷰" 3줄 세로 붕괴·상단 넘침, "삭제" 찌그러짐
AFTER — 제목 1줄, 버튼 한 줄 정렬, 삭제 danger
  • 제목 min-w-0 truncate, 컨트롤 shrink-0 + flex-wrap로 폭·폰트 변화를 흡수.
  • "삭제" 버튼을 danger(빨강)+휴지통 아이콘으로 — 파괴적 액션의 시각적 경고.
  • 모든 .btn에 focus-visible 링(키보드 접근성), 비활성 스타일 추가.

Help 언어 토글 before → after

BEFORE — 선택 언어가 파란 CTA 버튼처럼 보임
AFTER — 선택 상태(옅은 파랑 아웃라인)로 토글임을 명확히

3. 폰트 · 렌더링

BEFORE — 로고 🖼️ 두부(□), 비례폭 숫자
AFTER — 로고 아이콘, 등폭 숫자, antialiased
  • GPU 사용률·VRAM·온도·전력·step 등 주기적으로 갱신되는 수치에 tabular-nums → 폭 흔들림 제거.

  • .badge 11px → 12px, 깨지던 이모지 로고를 lucide Images 아이콘으로 교체.
